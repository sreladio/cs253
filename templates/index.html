Info

Hola, me llamo Carlos Eladio Fernández Ordoñez, y este es el sitio Web que he desarrollado para el curso Web Development de Udacity.

Está desarrollado con Python para el lado del servidor y Jinja2 como lenguaje de plantillas. Para el framework web he utilizado Webapp2, ya que permite un desarrollo a un nivel más bajo que Django.

Puedes ver y dercargar el código en Github.

Desarrollo

Las funcionalidades implementadas son:

  • Login y registro de nuevos usuarios (Signup):
    • Conexión HTTPs.
    • Validación de entrada de datos mediante expresiones regulares.
    • Las contraseñas son hasheadas con bcrypt antes de ser almacenadas en la BB.DD.
    • Las cookies son hasheadas mediante bcryp.
    • Verificación de las cookies cuando se carga una página.
    • Python Decorator para recursos con restricciones. Si un usuario que no está logedo intenta acceder a la página de edición de una wiki, se le redirecciona a la página de login. Si un usuario que ya está logeado intenta acceder a la página de login, se le redirecciona a la página de vienvenida.
  • Logout:
    • Borra la cookie y redirecciona a la página de login.
  • Blog:
    • New Post: permite añadir una nueva entrada al blog.
    • JSON: salida en formato JSON de las entradas del blog.
    • Memcache: la "front page" del blog es cacheada la primera vez que se accede o cuando se añade un nuevo post. También se cachea cada post de manera individual cuando se accede a través de su permalink. La opción Clear Memcache permite vaciar la caché.
  • Wiki
    • Necesario estar logeado para crear o editar una página.
    • Si se introduce una dirección que no existe en la barra de direcciones, se redireccionará automáticamente a la página de edición donde podemos añadir una nueva entrada a la wiki. En caso contrario (si la página existe), se mostrará su contenido.
    • En el historial de edición de una página, es posible ver y editar el contenido de versiones anteriores a la actual.

En cuanto a las ampliaciones y funcionalidades que quedan por hacer:

  • Markdown para permitir algunas etiquetas HTML en la edición de los post y de las páginas wiki evitando ataques XSS.
  • GetText para la internacionalización de la página.
  • CSS media queries para que la página se adapte tanto a dispositivos móviles como de escritorio.
